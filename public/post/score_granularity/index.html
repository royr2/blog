Optimising approval rates by creating segmented models

Credit risk series (Post #5)

Riddhiman Mar 25, 2023

Here’s a short post on why you should consider developing a segmented model when building a credit risk scorecards to help optimize approval rates.

I will be reusing code from a previous post of mine. Kindly navigate to this post for additional information. Note that smple datasets are available for download from here

Packages

#install.packages("pacman") ## Install if needed 
library(pacman)

# p_load automatically installs packages if needed
p_load(dplyr, magrittr, scales)

# Set seed
set.seed(1234)

Sample data

smple <- read.csv("../../../download/credit_sample.csv")

# Define target
codes <- c("Charged Off", "Does not meet the credit policy. Status:Charged Off")
smple %<>% mutate(bad_flag = ifelse(loan_status %in% codes, 1, 0))

# Some basic data cleaning
smple[is.na(smple)] <- -1
smple %<>% 
  
  # Remove cases where home ownership and payment plan are not reported
  filter(! home_ownership %in% c("", "NONE"),
         pymnt_plan != "") %>% 
  
  # Convert these two variables into factors
  mutate(home_ownership = factor(home_ownership), 
         pymnt_plan = factor(pymnt_plan))

Segments

Typically, credit risk scorecards would have segments like known goods, known bads, ever chaged off etc. For simplicity, I will use available FICO grade in the dataset to segment known goods and known bads.

smple %>% 
  group_by(grade) %>% 
  summarise(total = n(), bads = sum(bad_flag == 1)) %>% 
  mutate(event_rate = percent(bads / total))

For now, let’s simply keep those borrowers who have a FICO grade of "E", "F" & "G" in the known bads bucket and rest in the known goods bucket.

Model training

Let’s train two separate models. One to the entire population and one only to the known goods population.

# First train on the full population

# Train Test split
idx <- sample(1:nrow(smple), size = 0.7 * nrow(smple), replace = F)
train <- smple[idx,]
test <- smple[-idx,]

# Using a GLM model for simplicity
mdl_pop <- glm(
  formula = bad_flag ~ 
    loan_amnt + term + mths_since_last_delinq + total_pymnt + 
    home_ownership + acc_now_delinq + 
    inq_last_6mths + delinq_amnt + 
    mths_since_last_record + mths_since_recent_revol_delinq + 
    mths_since_last_major_derog + mths_since_recent_inq + 
    mths_since_recent_bc + num_accts_ever_120_pd,
  family = "binomial", 
  data = train
)

# Predict on population
smple$pred <- predict(mdl_pop, newdata = smple)

Now that we have a model, let’s assume an expected bad rate target of 2%. That is to say that I can only approve those customers who I believe have an expected default propensity of <=2%. Now the question is if that were the case, given the existing model, what would be my approval rate? Note that I need to maximise my approval rate (to better use my applicant funnel).

# Find the number of customers that can be approved such that 
# the cumulative bad rate is <= 2%
smple %>% 
  filter(! grade %in% c("E", "F", "G")) %>% 
  arrange(pred) %>% 
  mutate(c_bad_rate = cumsum(bad_flag)/cumsum(n())) %>% 
  filter(c_bad_rate <= tgt) %>% 
  summarise(`# Customers` = n()) %>% 
  mutate(`Approval Rate` = percent(`# Customers`/nrow(smple), accuracy = 0.1))
##   # Customers Approval Rate
## 1        4376         43.8%

Based on the unsegmented model, we are able to achieve an approval rate of 44.9%. Can we any better?

# Train only on the known good population

smple_kg <- smple %>% 
  filter(! grade %in% c("E", "F", "G"))

# Train Test split
idx <- sample(1:nrow(smple_kg), size = 0.7 * nrow(smple_kg), replace = F)
train <- smple_kg[idx,]
test <- smple_kg[-idx,]

# Using a GLM model for simplicity
mdl2 <- glm(
  formula = bad_flag ~ 
    loan_amnt + term + mths_since_last_delinq + total_pymnt + 
    home_ownership + acc_now_delinq + 
    inq_last_6mths + delinq_amnt + 
    mths_since_last_record + mths_since_recent_revol_delinq + 
    mths_since_last_major_derog + mths_since_recent_inq + 
    mths_since_recent_bc + num_accts_ever_120_pd,
  family = "binomial", 
  data = train
)

smple_kg$pred <- predict(mdl2, newdata = smple_kg)

smple_kg %>% 
  arrange(pred) %>% 
  mutate(c_bad_rate = cumsum(bad_flag)/cumsum(n())) %>% 
  filter(c_bad_rate <= tgt) %>% 
  summarise(`# Customers` = n()) %>% 
  mutate(`Approval Rate` = percent(`# Customers`/nrow(smple_kg), accuracy = 0.1))
##   # Customers Approval Rate
## 1        4380         47.7%
